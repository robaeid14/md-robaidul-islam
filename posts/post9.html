Section 9: Control Flow Graph (CFG)

What is a Control Flow Graph?

  • A Control Flow Graph (CFG) is a graphical representation of all possible paths that can be taken during the execution of a program.
  • It helps in understanding how control flows through a program's code.
  • Nodes represent basic blocks (a sequence of statements), and edges represent flow of control.

Components of CFG

  • Nodes: Represent a single or sequence of instructions (basic blocks).
  • Edges: Represent the control flow between nodes.
  • Entry/Exit Nodes: Indicate start and end of the control flow.

How to Construct a CFG

  1. Break code into basic blocks.
  2. Identify control flow (e.g., if, loops, switch).
  3. Draw nodes for each block and connect them using edges.

Example Code and CFG

if (x > 0) {
 x = x + 1;
} else {
 x = x - 1;
}
print(x);
  • Node 1: if (x > 0)
  • Node 2: x = x + 1
  • Node 3: x = x - 1
  • Node 4: print(x)
  • Edges: 1→2, 1→3, 2→4, 3→4

Importance in Testing

  • Used in white-box testing techniques.
  • Helps in path coverage, branch coverage, and condition coverage testing.
  • Improves understanding of program logic and control structure.

Applications of CFG

  • Compiler design and optimization
  • Static code analysis
  • Test case generation
  • Security analysis

Exam Point Summary

  • CFG is a graph showing flow of control in code.
  • It has nodes (blocks) and edges (control transitions).
  • Helps in measuring test adequacy and designing test cases.
  • Useful in white-box testing and code analysis.
