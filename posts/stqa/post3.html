Section 3: Testing and Development

Overview

Testing and development are two parallel and interdependent processes in software engineering. While development focuses on building the software, testing ensures that the built software is functioning correctly and meeting the requirements.

Relationship between Testing and Development

  • Complementary Processes:
    • Development produces the software artifacts (code, design, etc.).
    • Testing evaluates those artifacts to find defects and ensure quality.
  • Early Testing Principle:
    • Testing activities should begin as early as possible in the development life cycle to catch defects early and reduce cost.
    • Example: Reviewing requirements and design before coding.
  • Continuous Feedback Loop:
    • Test results provide feedback to developers for debugging, improving code quality, and refining design or requirements.
  • Shift-Left Testing:
    • Testing is integrated early in the development cycle, often during requirement analysis and design.
    • Helps in preventing bugs instead of just detecting them later.

Development Models and Testing Integration

  • Waterfall Model:
    • Testing is a separate phase after development.
    • Issues: Late defect detection, higher cost of fixing bugs.
  • V-Model (Verification & Validation Model):
    • Testing activities are mapped to each development stage.
    • Emphasizes early testing and traceability.
  • Agile Model:
    • Testing and development happen concurrently.
    • Focuses on continuous integration, frequent releases, and automated testing.

Developer vs Tester Roles

Aspect Developer Tester
Main Task Write code Test software
Objective Implement functionality Verify correctness
Mindset Creative (How to make it work) Destructive (How to break it)
Tools Used IDEs, compilers Test management, bug tracking tools
Scope Specific modules or features System-wide testing

Exam Point Summary

Testing and development are complementary processes.

  • Developers build the software, and testers evaluate it to ensure quality.
  • Testing should start early in the development cycle to reduce defect cost.
  • In models like V-Model or Agile, testing is integrated at every stage to improve feedback and reliability.
