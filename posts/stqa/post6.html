Section 6: Test Harness

Definition

A Test Harness is a collection of software and test data configured to test a program unit by running it under varying conditions and monitoring its behavior and outputs. It is mainly used in automated testing.

Main Components of a Test Harness

  • Test Execution Engine: Controls the running of test scripts.
  • Test Scripts: The automated test cases written to validate functionality.
  • Stubs and Drivers: Simulate parts of the system not yet developed or available.
  • Test Data: Input values for testing and their expected results.
  • Logging and Reporting Tools: Capture test results and generate reports.

Purpose of Using Test Harness

  • Automate repetitive testing tasks.
  • Ensure consistency in test execution.
  • Facilitate regression and performance testing.
  • Simulate external dependencies using stubs or mock objects.
  • Increase efficiency and coverage in testing.

Benefits of a Test Harness

  • Reduces manual effort and human error.
  • Faster execution and feedback during development.
  • Enables continuous integration and delivery (CI/CD).
  • Useful in both unit and integration testing.

Limitations

  • Initial setup can be time-consuming and complex.
  • Requires maintenance as application code changes.
  • Over-reliance on automation can miss usability or UI defects.

Example Use Case

Suppose you're testing a login feature, but the backend server isn’t ready. A test harness can simulate server responses using mock APIs while running automated login test scripts.

Common Tools Used

  • JUnit (Java)
  • TestNG
  • PyTest (Python)
  • Selenium (for web automation)
  • Postman (for API test harness)

Exam Point Summary

  • A test harness includes test scripts, stubs, drivers, and tools to automate software testing.
  • It is used to execute tests systematically and simulate missing parts of the system.
  • Improves efficiency, reusability, and automation in testing activities.
