Section 30: Problem Reporting & Corrective Action

1. What is Problem Reporting?

Problem reporting is the formal process of identifying, documenting, and communicating defects, issues, or anomalies found during testing. A well-managed problem reporting process helps ensure transparency, traceability, and resolution tracking.

2. Objectives

  • Identify and document defects clearly and consistently
  • Track the status of problems and fixes
  • Ensure accountability and traceability of defect resolutions
  • Communicate issues effectively among teams

3. Common Problem Report Components

Field Description
Summary Short description of the issue
Steps to Reproduce Clear steps to reproduce the defect
Expected Result What should have happened
Actual Result What actually happened
Severity Impact of the issue on system
Priority Importance and urgency of fix
Status Open, In Progress, Resolved, Closed, etc.
Environment Browser, OS, system version, etc.

4. Defect Lifecycle

  1. New
  2. Assigned
  3. Open / In Progress
  4. Fixed
  5. Retested
  6. Closed (or Reopened)

5. Corrective Action Process

Corrective action refers to steps taken to eliminate the root cause of a problem or defect and prevent recurrence. This process usually involves:

  • Analyzing the defect
  • Identifying the root cause
  • Implementing the fix
  • Testing the fix
  • Updating documentation or code standards

6. Example: Bug Report Format

Bug ID: #1014
Summary: Login button does not respond
Steps to Reproduce:
  1. Navigate to Login Page
  2. Enter valid credentials
  3. Click "Login"
Expected Result: User is redirected to dashboard
Actual Result: Nothing happens
Severity: High
Priority: Urgent
Status: Open
Environment: Chrome 114, Windows 10

7. Benefits of Effective Problem Reporting & Correction

  • Reduces defect leakage to production
  • Improves software quality and customer satisfaction
  • Helps in audit and compliance
  • Encourages accountability within teams
