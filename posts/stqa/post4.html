Section 4: Test Case

Definition

A test case is a set of conditions or variables under which a tester determines whether a software system or one of its features is working as intended.

It includes input data, execution steps, and the expected outcome to validate a specific software feature or function.

Key Components of a Test Case

  • Test Case ID: Unique identifier for tracking.
  • Test Description: Purpose of the test.
  • Preconditions: What must be true before running the test.
  • Test Steps: Actions to perform.
  • Test Data: Input values for the test.
  • Expected Result: What should happen.
  • Actual Result: What actually happened during execution.
  • Status: Pass/Fail based on comparison of expected vs actual result.

Types of Test Cases

  • Positive Test Case: Tests system behavior with valid input.
  • Negative Test Case: Tests system response to invalid or unexpected input.
  • Boundary Test Case: Focuses on edge values (e.g., min/max limits).
  • Functional Test Case: Validates specific features against requirements.
  • Non-Functional Test Case: Tests performance, usability, security, etc.

Why Test Cases Are Important

  • Ensure systematic testing and traceability.
  • Help reproduce bugs and errors clearly.
  • Enable consistent regression testing.
  • Support test automation with clear input/output.
  • Provide documentation of what has been tested.

Sample Test Case Format

  • ID: TC_01
  • Description: Verify login with valid credentials
  • Precondition: User is on the login page
  • Steps:
    1. Enter valid username
    2. Enter valid password
    3. Click Login
  • Test Data: username: testuser, password: test123
  • Expected Result: Redirect to dashboard
  • Actual Result: Redirect to dashboard
  • Status: Pass

Best Practices for Writing Test Cases

  • Keep them simple and clear.
  • Use consistent naming conventions.
  • Link to requirement IDs (if applicable).
  • Include both positive and negative scenarios.
  • Review and update regularly.

Exam Point Summary

  • A test case defines how to test a software function using specific inputs and expected outcomes.
  • It includes ID, description, steps, expected/actual result, and status.
  • Test cases ensure thorough, repeatable, and documented testing.
  • Both functional and non-functional scenarios should be covered.
