Section 5: Test Execution

Definition

Test Execution is the process of running the prepared test cases on the software application to validate its behavior and functionality. It is a critical phase in the software testing life cycle.

Steps in Test Execution

  1. Test Environment Validation: Ensure all components (server, database, etc.) are set up and ready.
  2. Execute Test Cases: Run each test manually or using automation tools.
  3. Log Results: Record actual outcomes of the test execution.
  4. Report Defects: Log any deviations or failures as bugs using tracking tools.
  5. Retesting: Re-run failed test cases after the defect is fixed.
  6. Regression Testing: Ensure new changes have not broken existing functionality.

Inputs Required for Test Execution

  • Approved Test Cases and Scripts
  • Test Data
  • Configured Test Environment
  • Test Plan
  • Defect Management Tool Access

Test Execution Outputs

  • Executed Test Logs
  • Defect Reports
  • Pass/Fail Status for Each Test Case
  • Updated Test Metrics

Common Test Execution Metrics

  • Pass Percentage: (Passed cases / Total cases) × 100
  • Defect Density: Defects per module or per lines of code
  • Test Execution Progress: % of test cases executed
  • Re-test Rate: How often test cases are re-executed due to fixes

Best Practices

  • Prioritize critical test cases first.
  • Use automation for regression testing.
  • Continuously track and analyze test results.
  • Maintain proper logs and documentation.
  • Communicate with developers for faster resolution of issues.

Challenges During Test Execution

  • Unstable test environment
  • Incomplete requirements or test data
  • Tool or script failures
  • Delays in defect fixing by developers

Exam Point Summary

  • Test execution involves running test cases and comparing actual results with expected outcomes.
  • Includes logging results, reporting defects, retesting, and regression testing.
  • Generates metrics and insights to evaluate software quality.
  • Relies on a prepared environment, complete test data, and communication with development teams.
