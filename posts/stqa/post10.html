Section 10: Errors, Faults, and Failures

Introduction

Errors, faults, and failures are fundamental concepts in software testing and quality assurance. Understanding the relationship between them helps in identifying, analyzing, and correcting problems in software systems.

Definitions

  • Error (Mistake): A human action that produces an incorrect result. Usually made by developers, designers, or testers.
  • Fault (Bug or Defect): The incorrect decision or code written due to an error. It exists in the software system or code.
  • Failure: The observable incorrect behavior or output of the software during execution due to a fault.

Relationship Between Them

  • Error → Fault → Failure
  • An error made by a human results in a fault in the code.
  • When that faulty code executes, it can lead to a failure (visible problem).

Example

  • Developer incorrectly writes if (a = b) instead of if (a == b) → Error
  • This causes the program to assign value instead of comparing → Fault
  • Program behaves unexpectedly during execution → Failure

Classification of Faults

  • Syntax Fault: Errors in grammar or structure of the code (e.g., missing semicolon).
  • Logical Fault: Code is syntactically correct but implements wrong logic.
  • Computation Fault: Incorrect calculation or processing.
  • Interface Fault: Incorrect interaction with external systems or components.
  • Initialization Fault: Variables or states not initialized correctly.

Importance in Testing

  • Helps testers understand where and why issues occur.
  • Guides in creating effective test cases.
  • Improves debugging and quality assurance.

Exam Point Summary

  • Error: Human mistake during development.
  • Fault: Incorrect code resulting from an error.
  • Failure: Incorrect software behavior during execution.
  • Error leads to fault, and fault causes failure during execution.
