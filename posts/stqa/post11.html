Section 11: Types of Testing

Definition

Types of software testing refer to the various approaches and strategies used to verify and validate software quality, functionality, performance, and usability. Different testing types are applied based on project needs, development model, and objectives.

Main Types of Testing

  1. Unit Testing:

    Tests individual units or components of a software system (e.g., functions, classes).

  2. Integration Testing:

    Ensures different modules or units work together correctly.

  3. System Testing:

    Tests the complete integrated system to verify it meets specified requirements.

  4. Acceptance Testing:

    Validates the software with end-user requirements. Usually performed by users or clients.

Other Important Types

  • Regression Testing: Ensures that recent code changes haven't affected existing functionality.
  • Smoke Testing: Basic tests to check if the build is stable enough for further testing.
  • Sanity Testing: Narrow regression tests focusing on specific functionalities.
  • Performance Testing: Checks how the software performs under load or stress.
  • Security Testing: Evaluates the system's resistance to threats and vulnerabilities.
  • Usability Testing: Assesses user experience, ease of use, and interface friendliness.
  • Compatibility Testing: Verifies functionality across different environments, OSs, and browsers.
  • Alpha Testing: Conducted internally before releasing to users.
  • Beta Testing: Conducted by selected users in a real-world environment.

Classification Based on Approach

  • Manual Testing: Testing without automation tools. Testers execute test cases manually.
  • Automated Testing: Uses tools or scripts to perform testing tasks.

White Box vs Black Box vs Grey Box

  • White Box Testing: Tests internal logic or code structure.
  • Black Box Testing: Tests functionality without looking at the internal structure.
  • Grey Box Testing: Combines both approaches, having partial knowledge of internal code.

Exam Point Summary

  • Types of testing are chosen based on testing level, approach, and objectives.
  • Main types include Unit, Integration, System, and Acceptance testing.
  • Other types include Regression, Performance, Security, Usability, and Compatibility testing.
  • Testing may be manual or automated, and categorized as black box, white box, or grey box.
