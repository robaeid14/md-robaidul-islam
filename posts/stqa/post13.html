Section 13: Test Adequacy Assessment

Definition

Test adequacy assessment determines how sufficient and effective a set of test cases is in testing a software application. It helps in understanding how well the software has been tested and whether additional tests are needed.

Adequacy Criteria

Adequacy criteria are the rules or requirements that a test set must satisfy to be considered adequate. Common adequacy criteria include:

  • Statement coverage
  • Branch coverage
  • Path coverage
  • Condition coverage

Control Flow Based Criteria

This criterion evaluates how well the test cases exercise the control flow (decision points, loops, etc.) of a program. Examples include:

  • Statement Coverage: Ensures all statements are executed at least once.
  • Branch Coverage: Ensures all branches (true/false outcomes) of decision points are tested.
  • Path Coverage: Ensures all possible paths in the control flow graph are covered.

Data Flow Based Criteria

This criterion assesses the interaction of program variables throughout the execution. It includes:

  • Definition-Use (DU) Pairs: Covers all combinations of variable definitions and their corresponding uses.
  • All-Defs and All-Uses Criteria: Requires that each variableâ€™s definition is tested with all its uses (computational or predicate).

Mutation Based Criteria

This criterion uses small changes (mutants) in the program code to test whether the test cases can detect them. If a test case fails when a mutant is introduced, the test is considered strong.

  • Mutation Score: Percentage of mutants detected by the test cases.
  • Equivalent Mutants: Mutants that do not change the behavior of the program and cannot be detected.

Adequacy as a Stopping Criterion

Test adequacy can serve as a metric to decide when to stop testing. If a certain adequacy level (e.g., 90% branch coverage) is reached, further testing might not be necessary.

Adequacy as a Tool for Test Enhancement

By analyzing adequacy metrics, testers can identify untested areas and add more test cases to improve coverage. This leads to better quality and higher reliability of the software.

Exam Point Summary

  • Test adequacy assesses the sufficiency of a test suite.
  • Common criteria: control flow, data flow, and mutation-based.
  • Used as a guideline to stop or enhance testing efforts.
