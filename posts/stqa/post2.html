Section 2: Test Process

Definition

The software test process is a systematic sequence of activities carried out to ensure the quality of software through testing. It defines how testing should be planned, executed, and evaluated throughout the software development life cycle (SDLC).

Phases of the Software Testing Process

  1. Test Planning
    • Define the scope, objectives, resources, schedule, and approach of testing.
    • Create the Test Plan document.
    • Identify risks, tools, and responsibilities.
  2. Test Design
    • Identify test conditions, test cases, and test data based on requirements and design documents.
    • Use test design techniques such as equivalence partitioning, boundary value analysis, etc.
  3. Test Environment Setup
    • Prepare the hardware and software environment where testing will be performed.
    • Install required tools, databases, and application components.
  4. Test Implementation
    • Develop and organize test cases and test scripts.
    • Arrange test data and make test cases ready for execution.
  5. Test Execution
    • Run the test cases.
    • Log test results and record any defects/bugs found.
    • Repeat failed tests after fixing issues (re-testing).
  6. Defect Reporting and Tracking
    • Report issues found during testing using a bug tracking tool (e.g., Bugzilla, JIRA).
    • Track the status of defects until they are resolved and retested.
  7. Test Closure
    • Evaluate whether testing goals were met.
    • Create test summary reports.
    • Archive test artifacts and analyze lessons learned.

Test Process Flow (Simplified)

Test Planning → Test Design → Environment Setup → Test Execution → Defect Tracking → Test Closure

Key Outputs / Artifacts

  • Test Plan
  • Test Cases
  • Test Scripts
  • Test Data
  • Defect Reports
  • Test Summary Report

Exam Point Summary

The software test process involves the following phases:

  • Test Planning – defining strategy and scope.
  • Test Design – creating test cases and data.
  • Environment Setup – preparing system for testing.
  • Test Execution – running tests and logging results.
  • Defect Reporting – identifying and tracking bugs.
  • Test Closure – finalizing and reporting outcomes.
