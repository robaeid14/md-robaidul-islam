Section 8: Test Adequacy

Definition

  • Test adequacy is a measure of how well a test suite tests a program. It assesses the completeness and effectiveness of testing based on certain criteria.

Objectives of Test Adequacy

  • To evaluate the sufficiency of test cases.
  • To ensure that critical parts of code are tested.
  • To define measurable goals for testing.

Adequacy Criteria

  • Statement Coverage: Checks if all executable statements have been tested.
  • Branch Coverage: Ensures each possible branch (if/else) is tested.
  • Condition Coverage: Verifies each boolean sub-expression evaluates both true and false.
  • Path Coverage: Tests all possible paths through a given part of the code.
  • Data Flow Coverage: Focuses on variable definitions and their uses.
  • Mutation Testing: Checks if test cases can detect small changes (mutants) in the program.

Why Test Adequacy is Important

  • Improves confidence in the quality of software.
  • Reveals gaps in testing and potential undetected bugs.
  • Helps determine when testing is enough.

Challenges

  • Achieving 100% coverage is often impractical.
  • Some paths or conditions may be infeasible to test.
  • High coverage doesn’t always guarantee defect-free software.

When to Use Adequacy Criteria

  • To assess test suite quality in regulated industries (e.g., medical, aviation).
  • During test optimization and review phases.
  • For critical systems where fault tolerance is important.

Exam Point Summary

  • Test adequacy measures the completeness of testing.
  • Common adequacy criteria include statement, branch, condition, and path coverage.
  • It helps ensure sufficient testing and identify weak spots in the test suite.
  • However, 100% coverage does not guarantee zero defects.
